What to build next

Product Direction

What will make this feel real

  • A saved dashboard model with name, owner, audience, and layout JSON
  • Widget types for number, line, bar, table, and text blocks
  • Data sources powered by Reports first, custom endpoints second
  • A quiet UI with strong spacing, clear hierarchy, and few decisions per screen
Suggested Build Order
  1. Create DocTypes for dashboards, widgets, and data source definitions.
  2. Expose data through whitelisted methods or report-backed APIs.
  3. Render a drag-and-drop page experience for editing layouts.
  4. Ship a polished viewer mode for end users and stakeholders.
Why Frappe Fits

Frappe already gives you auth, permissions, metadata, reports, and APIs. That means the hard part here is not infrastructure. It is the editing experience and the quality of the visual system.
